## 💡 The Concept — The Junior's Edge

> While competitors focus on English-speaking markets, charge for premium features, or are locked to desktop — **Hunter** was built from day one for the **Israeli student & junior developer** entering the tech market.

| Problem | Hunter's Solution |
|---|---|
| 🌐 Tools are in English | 🇮🇱 Built for the Israeli market (Drushim, AllJobs, etc.) |
| 💸 Premium features cost money | ✅ All features free |
| 🖥️ Desktop-only experience | 📱 Mobile-first, built for your pocket |
| 📋 Too many required fields | ⚡ Speed Apply — one tap, zero typing |

```
├──────────────────┬──────────────────┬───────────────────────┤
│   Presentation   │    Data Layer    │      Services         │
├──────────────────┼──────────────────┼───────────────────────┤
│  Kotlin          │  Firebase Auth   │  AlarmManager         │
│  ViewBinding     │  Firestore DB    │  Intent Filters       │
│  Fragments       │  Cloud Storage   │  Deep Links           │
│  RecyclerView    │  Real-time Sync  │  Share Intents        │
│  Material 3      │                  │  BroadcastReceiver    │
│  Vico Charts     │                  │                       │
│  Montserrat Font │                  │                       │
└──────────────────┴──────────────────┴───────────────────────┘
```

</div>

### Architecture Highlights

**🔗 Speed Apply — Deep Link Engine**
The `SpeedApplyActivity` intercepts shared URLs from **LinkedIn**, **Indeed**, **Drushim**, and **AllJobs** directly from the Android share sheet — turning a job post into a logged application in seconds.

**📂 CV Versioning — The Killer Feature**
Upload multiple PDF versions, tag them (`Fullstack`, `Backend`, `Data`), and the app remembers which version went to which company. Stored in Firebase Storage, managed through a clean card-based UI.

**🤝 Interview Intelligence — Social Layer**
Community-shared interview questions are stored in Firestore and searchable by company. This is the feature that transforms Hunter from a personal tracker into a **collaborative platform**.

**⏰ Smart Reminders**
`AlarmManager` + `BroadcastReceiver` ensures that scheduled interviews trigger local notifications — even when the app is closed.

## 📦 Firebase Architecture

```
firestore/
├── job_applications/     ← Per-user job pipeline (private)
│   └── {jobId}/          ← title, company, status, CV version sent...
├── interviews/           ← Scheduled interviews with reminders
│   └── {interviewId}/
├── interview_questions/  ← 🌐 SHARED — community interview intel
│   └── {questionId}/     ← company, question, anonymous flag...
├── cv_files/             ← CV metadata (PDF stored in Storage)
└── users/                ← User profile data

storage/
└── cv_files/{userId}/    ← Actual PDF files (secured by Auth rules)
```
